Redesigning the user interface on Outlook Mobile Application - a UX case study


Written by: Susie Nguyen
Last Updated: February 3, 2019

With the development of digital technology and rise in the use of the hand held devices, the number of various mobile applications available in the market are increasing tremendously. Understanding the concepts of user experience and user interface design is significant in the process of creating and implementing a mobile application.

Microsoft Outlook is a popular email application by Microsoft available both on desktop and mobile devices. Despite being widely used, the application is still lacking in certain usability and UX dimensions. In particular, the user interface for the Outlook Calendar on its mobile application needs considerable improvements.

Identifying the users and their goals

The primary users of the Outlook Mobile App often consist of individuals who have already created an Outlook account on the Desktop version of the email application or individuals who prefer accessing their email using a mobile device.

The goals of the users of the Outlook Mobile App are:

  • Sending/Receiving emails
  • Receiving instant email notifications
  • Utilizing Outlook Calendar
    • Getting reminders on the calendar events attached in the emails
    • Getting reminders on the calendar
    • Use it as a digital weekly/monthly/yearly planner
    • Sorting the events on the calendar
    • Use it to send a Calendar Invite/Use it for scheduling appointments
  • Accessing Office 365 document attachments
  • Sorting emails

Design Problem Statement

Despite being a great email mobile application and time management tool, Outlook Mobile Application should make improvements towards the way it is achieving those user goals. Specifically, the user interface design of the Outlook Calendar on its mobile application can often be unclear, inefficient and error-prone. The main pain point of the Outlook Calendar is that the design features of this application do not communicate well with their functionality as it does not exhibit proper visual and physical properties.

Design Question: "How might we improve the experience of individuals utilizing Outlook Calendar on their mobile device?"

In contrast to the desktop version of the email application, there is no way on the mobile app to create an event with a different colored tag. Although the mobile version does sync with the actions done on the desktop, having to do this on the desktop in order to view it on the mobile diminishes the purpose of installing the mobile application.

Even on the desktop version, it is unintuitive as you would have to go through multiple steps in order to create an event with a different colored tag. First, you would have to “Add a calendar”, choose the color of the tag you want for the events on that calendar. Next, add new events to that calendar. Then, you would have to select the calendars you want to see on the default view of the Calendar feature. Finally, only when you have selected the calendars you want to see on your main view will the your Outlook Calendar show the events categorized by color. As demonstrated by the complex process of sorting events on the Calendar feature of the Outlook mobile app, the design features of this application does not communicate well with their functionality as it does not exhibit proper visual and physical properties.
