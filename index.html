Redesigning the user interface of Outlook Mobile Application - a UX case study


Written by: Susie Nguyen
Last Updated: February 3, 2019

With the development of digital technology and rise in the use of the hand held devices, the number of various mobile applications available in the market are increasing tremendously. Understanding the concepts of user experience and user interface design is significant in the process of creating and implementing a mobile application.

Microsoft Outlook is a popular email application by Microsoft available both on desktop and mobile devices. Despite being widely used, the application still lacks in certain dimensions of usability and user experience. In particular, the user interface for the Outlook Calendar on its mobile application needs considerable improvements.

Identifying the users and their goals

The primary users of the Outlook Mobile App often consist of individuals who have already created an Outlook account on the Desktop version of the email application or individuals who prefer accessing their email using a mobile device.

The goals of the users of the Outlook Mobile App are:

  • Sending/Receiving emails
  • Receiving instant email notifications
  • Utilizing Outlook Calendar
    • Getting reminders on the calendar events attached in the emails
    • Getting reminders on the calendar
    • Use it as a digital weekly/monthly/yearly planner
    • Sorting the events on the calendar
    • Use it to send a Calendar Invite/Use it for scheduling appointments
  • Accessing Office 365 document attachments
  • Sorting emails

App Analysis

Despite being a great email mobile application and time management tool, Outlook Mobile Application should make improvements towards the way it is achieving those user goals. Specifically, the user interface design of the Outlook Calendar on its mobile application can often be unclear, inefficient and error-prone. The interface needs improvement in the following usability dimensions: standardization, visibility, and natural mapping.

Standardization

The mobile version of Outlook Calendar is inconsistent with the features available on its desktop version as well as its visual implementation. For instance, in contrast to the desktop version of the email application, the mobile version has does not provide any method for the user to create an event with a different colored tag. Although the mobile version does sync with the actions done on the desktop, having to do this on the desktop in order to view it on the mobile device diminishes the purpose of installing the mobile application.

Even on the desktop version, sorting Outlok Calendar events and categorizing them is unintuitive as you would have to go through multiple complicated steps in order to achieve that complete and color-coded view of events. First, you would have to click on “Add a calendar”, then choose the color of the tag you want for the events on that calendar. Next, add new events to that calendar. Afterward, you would have to select the calendars you want to see on the default view of the Calendar feature. Finally, only when you have selected the calendars you want to see on your main view, will your Outlook Calendar show the view of all the events with different colored tags based on the calendars created. This process requires too many steps just to be able to give each event category a different color.

Visibility and Natural Mapping

Beside being inconsistent with the "Add Calendar" option from the desktop implementation, Mobile Outlook Calendar does not exhibit good visibility and natural mapping. As you can see, an index pane is used to store the secondary settings for the calendar. Even though index panes work well most of the time for secondary settings, the ability to change color for the Calendar should be considered as one of the primary tasks and should be displayed on the same control panel as choosing an icon for an event. Instead, this feature is displayed on another the pop-up menu within that index pane. This mapping can be unintuitve and confusing for new users and weakens the visibility of available functions.



Design Problem Statement

As demonstrated by this complicated and long process of categorizing events on Outlook Calendar by color, we can conclude that the main pain point of this application is that its design features do not communicate well with its functionalitise as they does not exhibit proper visual and physical properties.

Design Question: "How might we improve the experience of individuals utilizing Outlook Calendar on their mobile device?"

User Research

The participants of this case study are twenty-two college students from James Madison University who are all also users of Microsoft Outlook email application. They were contacted by email and through social media platforms, and were asked to fill out a Google Survey. I would like to acknowledge that due to the small scope of the participants there might be limitations to this case study, however, there is no doubt that the data collected from these twenty-two participants will definitely gather some valuable insights.

As demonstarted by the results of the survey, most participants use Outlook Mobile App for getting an instant access to their emails as well as utilizing Outlook Calendar. When asked to rate their experience interacting with the current Outlook Calendar interface (on a scale from 1-5), more than half rated their experience as something lower than a four.

Thirteen out of twenty-two participants have agreed that Outlook Calendar needs major revisions on its functionality as a digital planner and task management. In response to the question asking to describe why they think those features need improvements, one of the participants (Tam Nguyen) said "User interface can be improved to me more intuitive and simplified!".

The Solution

Based on the feedback from the participants and the results of the Google Survey as well as the analysis of UX and usability dimensions of the app, I have came up with the new design implementation for the sorting features of Outlook Calendar.

New Event Categorization Control Panel

In contrast to the current implementation, this design solution gives the users a two-in-one step of sorting the calendar events by category. By having both - the option to select a color as well as the option to choose an icon for the calendar event - the process of creating an event and giving it a category is significantly simplified. By default, the color of a calendar event will show up as blue (Outlook's primary color).

This design implementation eleminates the inefficiency in sorting the events and the need to switch to the desktop version in order to achieve that goal. The mobile application has become just as useful as the desktop version. Now, you can stay organized with Outlook even just using the mobile version.
